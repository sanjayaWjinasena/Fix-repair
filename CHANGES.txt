CURRENT WORK (not-under-warranty-with-serial-no branch)
═══════════════════════════════════════════════════════════════

GOAL
----
Add support for "Normal Repair (With Serial No)" / "Not Under Warranty"
repair type. These repairs skip the RUG approval flow and instead send
the quotation directly to the customer via Send by Email.


10. PLAN INTERVENTION BUTTON FOR NON-WARRANTY REPAIRS
    File: models/helpdesk_ticket.py (_get_view)
    - Old condition required x_studio_valid_return for ALL repairs.
    - Warranty (RUG) repairs need a return picking to confirm the item
      is physically at the factory.
    - Non-warranty customers bring the item directly — no return picking.
    - New condition: (x_studio_rug_repair and not x_studio_valid_return)
      so non-warranty repairs show Plan Intervention without a return.

11. NOT UNDER WARRANTY QUOTATION TYPE — SEND BY EMAIL BUTTON
    File: models/sale_order.py (_get_view)
    - Studio has disabled all action_quotation_send buttons via
      `or (state not in ['False'])` (always-true kill switch).
    - Injected a clean Send by Email button via lxml in _get_view:
        name="action_quotation_send"
        invisible="x_studio_quotation_type != 'Not Under Warranty'
                   or state not in ('draft', 'sent')"
    - Request RUG Approval (1980) already gates on type == 'Repair',
      so it hides automatically for 'Not Under Warranty'.
    - Confirm button guard only applies to type == 'Repair', so it
      works normally for 'Not Under Warranty'.

12. ADDING 'NOT UNDER WARRANTY' AS A VALID SELECTION VALUE
    File: models/sale_order.py (_ensure_not_under_warranty_selection)
    File: data/fix_repair_data.xml (<function> call)
    File: __manifest__.py (data file registered)

    Problem history:
      - selection_add in Python fails on Studio fields (no Python base).
      - _auto_init override not called during upgrade of fix_repair
        (sale.order is owned by the 'sale' module, not fix_repair).
      - Raw SQL on ir_model_fields failed: no 'selection' column in Odoo 17.

    Final solution:
      - _ensure_not_under_warranty_selection() uses ir.model.fields.selection
        ORM to search for the value and create it if absent.
      - Called from data/fix_repair_data.xml (<function>) on every
        install/upgrade.
      - Also called inline in _fsm_ensure_sale_order before writing the
        value, so it self-heals even if the data file hasn't run yet.

13. QUOTATION TYPE FIELD — READONLY CONDITION RELAXED
    File: models/sale_order.py (_get_view)
    - Old: readonly when type == 'Repair' OR task linked OR not draft/sent.
    - New: readonly only when task linked OR not draft/sent.
    - Allows switching between Repair and Not Under Warranty in draft/sent
      state as long as no FSM task has been created yet.

14. AUTO-SET SO TYPE FROM TICKET FLAG ON PLAN INTERVENTION
    File: models/project_task.py (_fsm_ensure_sale_order)
    - When Plan Intervention creates the FSM sale order, reads the ticket's
      x_studio_normal_repair_with_serial_no flag:
        True  → x_studio_quotation_type = 'Not Under Warranty'
        False → x_studio_quotation_type = 'Repair'
    - Ensures the selection value exists before writing (self-healing guard).

    Ticket fields that distinguish the 4 repair types:
      x_studio_rug_repair                    → Warranty / RUG repair
      x_studio_normal_repair_with_serial_no  → Not Under Warranty (with SN)
      x_studio_normal_repair_without_serial_no → Normal repair (no SN)
      x_studio_quick_repair_status           → Quick repair

═══════════════════════════════════════════════════════════════
KEY DESIGN DECISIONS
═══════════════════════════════════════════════════════════════

- Studio fields (x_studio_*) cannot be extended with selection_add in Python
  because they have no Python base definition. Use ir.model.fields.selection
  ORM instead.

- _auto_init in _inherit classes is NOT called during module upgrade for
  models owned by other modules. Use XML <function> data calls instead.

- Stage transitions for repair tickets are handled in two places:
    Path A: stock_picking._action_done (picking validation → ticket stage)
    Path B: stock_picking._action_done (return picking → Handed Over)
  NOT in Mark as Done (action_fsm_validate) — that only gates the button.

- Confirm button on Repair SOs is blocked until x_studio_rug_approved=True.
  Not Under Warranty SOs skip this guard entirely (no RUG flow).

- Send by Email on sale.order is disabled for all types by Studio via
  `or (state not in ['False'])`. Injected fresh via _get_view for
  Not Under Warranty type only.
